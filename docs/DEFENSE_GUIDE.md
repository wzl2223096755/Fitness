## 1. 项目概述

### 1.1 项目背景

随着健身运动的普及，科学训练和恢复管理变得越来越重要。本系统旨在帮助健身爱好者：
- 科学记录和分析训练数据
- 评估身体恢复状态
- 获取个性化训练建议
- 追踪营养摄入

### 1.2 核心功能

| 功能模块 | 主要功能 | 技术实现 |
|----------|----------|----------|
| 用户管理 | 注册、登录、个人信息 | JWT + BCrypt |
| 训练数据 | 记录、统计、分析 | JPA + ECharts |
| 负荷计算 | 训练量、1RM估算 | 算法实现 |
| 恢复评估 | 评分、建议生成 | 多因素算法 |
| 营养追踪 | 饮食记录、卡路里 | CRUD + 统计 |
| 数据可视化 | 图表、趋势分析 | ECharts |
| 管理后台 | 监控、导出、管理 | Actuator + POI |

### 1.3 技术架构

- **前端**: Vue 3 + Element Plus + ECharts + Pinia
- **后端**: Spring Boot 3.2.5 + Spring Security + JPA
- **数据库**: MySQL 8.0 / H2
- **缓存**: Caffeine + Redis
- **部署**: Docker + Nginx

## 3. 核心算法说明

### 3.1 训练量计算

```
训练量 (Volume) = 重量 × 组数 × 次数

示例：
- 杠铃卧推 60kg × 4组 × 10次 = 2400kg
```

**代码实现**：
```java
public Double calculateVolume(Double weight, Integer sets, Integer reps) {
    if (weight == null || sets == null || reps == null) {
        return 0.0;
    }
    return weight * sets * reps;
}
```

### 3.2 1RM（最大重量）估算

使用 **Epley公式**：

```
1RM = 重量 × (1 + 次数/30)

示例：
- 60kg × 10次 → 1RM = 60 × (1 + 10/30) = 80kg
```

**代码实现**：
```java
public Double calculateOneRepMax(Double weight, Integer reps) {
    if (weight == null || reps == null || reps <= 0) {
        return weight;
    }
    if (reps == 1) {
        return weight;
    }
    // Epley公式
    return weight * (1 + reps / 30.0);
}
```

### 3.3 恢复评分算法

**多因素加权评分**：

```
基础分 = 50分

睡眠因素：
  - 睡眠时长加分 = (sleepHours - 6) × 5  (6小时为基准)
  - 睡眠质量加分 = (sleepQuality - 5) × 3  (5分为基准)

身体因素：
  - 肌肉酸痛扣分 = muscleSoreness × 3
  - 压力水平扣分 = stressLevel × 2

生理指标：
  - HRV加分 = (hrv - 50) × 0.5  (50为基准)
  - 静息心率扣分 = (restingHR - 60) × 0.3  (60为基准)

最终评分 = 基础分 + 各项加减分（限制在0-100范围）
```

**代码实现**：
```java
public int calculateRecoveryScore(RecoveryData data) {
    double score = 50.0; // 基础分
    
    // 睡眠因素
    if (data.getSleepHours() != null) {
        score += (data.getSleepHours() - 6) * 5;
    }
    if (data.getSleepQuality() != null) {
        score += (data.getSleepQuality() - 5) * 3;
    }
    
    // 身体因素
    if (data.getMuscleSoreness() != null) {
        score -= data.getMuscleSoreness() * 3;
    }
    if (data.getStressLevel() != null) {
        score -= data.getStressLevel() * 2;
    }
    
    // 生理指标
    if (data.getHeartRateVariability() != null) {
        score += (data.getHeartRateVariability() - 50) * 0.5;
    }
    
    // 限制范围
    return (int) Math.max(0, Math.min(100, score));
}
```

### 3.4 恢复状态分类

| 评分范围 | 状态 | 建议 |
|----------|------|------|
| 80-100 | 恢复良好 | 可进行高强度训练 |
| 60-79 | 恢复一般 | 建议中等强度训练 |
| 40-59 | 轻度疲劳 | 建议低强度训练或休息 |
| 20-39 | 中度疲劳 | 建议休息或轻度活动 |
| 0-19 | 严重疲劳 | 强烈建议休息 |

## 4. 技术亮点

### 4.1 架构设计亮点

| 亮点 | 说明 | 价值 |

| **模块化架构** | 采用DDD思想，5个独立业务模块 | 高内聚低耦合，易于维护扩展 |
| **事件驱动** | 模块间通过Spring Events通信 | 解耦模块依赖，支持异步处理 |

### 4.2 安全设计亮点

| 亮点 | 说明 | 价值 |
|------|------|------|
| **JWT认证** | 无状态Token认证 | 支持分布式部署 |
| **BCrypt加密** | 密码安全存储 | 防止密码泄露 |
| **XSS防护** | 输入过滤和输出编码 | 防止脚本注入 |

## 6. 常见问题准备

### Q1: 为什么选择Spring Boot + Vue技术栈？

**回答要点**：
- Spring Boot是Java生态最流行的Web框架，企业应用广泛
- Vue 3是渐进式框架，学习曲线平缓，生态丰富
- 前后端分离架构，便于团队协作和独立部署
- 技术成熟稳定，社区活跃，文档完善

### Q2: 系统的安全性如何保证？

**回答要点**：
- JWT无状态认证，支持Token刷新
- BCrypt密码加密，防止彩虹表攻击
- XSS过滤器防止脚本注入
- CORS配置限制跨域访问
- API限流防止恶意请求
- 审计日志记录敏感操作

### Q3: 恢复评分算法的依据是什么？

**回答要点**：
- 参考运动科学研究，睡眠是恢复的关键因素
- 肌肉酸痛和压力水平反映身体负荷
- HRV（心率变异性）是恢复状态的重要指标
- 多因素加权综合评估，比单一指标更准确

### Q4: 系统如何处理高并发？

**回答要点**：
- 多级缓存减少数据库压力
- HikariCP连接池优化数据库连接
- 无状态设计支持水平扩展
- 异步事件处理解耦业务逻辑

### Q5: 项目的创新点是什么？

**回答要点**：
- 模块化架构设计，采用DDD思想
- 多因素恢复评分算法
- 事件驱动的模块间通信
- 完整的监控和可观测性体系
- 全面的测试覆盖（单元+属性+E2E）

### Q6: 未来可以如何扩展？

**回答要点**：
- 接入智能穿戴设备数据
- 引入机器学习优化训练建议
- 添加社交功能（训练分享、排行榜）
- 支持更多训练类型（瑜伽、游泳等）
- 移动端App开发
